# Integrations

SalesFlow uses generic target names so teams can map it to their own stack.

## Generic Targets

| SalesFlow target | Typical implementation |
| --- | --- |
| `sales_knowledge_base` | CRM, knowledge base, sales memory, customer island |
| `personal_knowledge_base` | personal memory, team learning, reusable judgment |
| `lightweight_tracker` | spreadsheet, Airtable, Lark/Feishu Base, CRM task view |
| `local_archive` | local folder, Drive, SharePoint, document repository |
| `quotation_workflow` | quoting tool, CPQ, workbook, pricing review process |

## Optional Product Pairing

- [KnowSales](https://www.knowsales.ai): sales memory and knowledge base for customer profiles, activities, product knowledge, objection cards, and competitor intelligence.
- [KnowMine](https://knowmine.ai): AI-native personal knowledge base for reusable judgment, preferences, and process memory.

SalesFlow does not require these products. They are useful when a team wants a persistent memory layer behind the workflow.
