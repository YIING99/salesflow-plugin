# Daily Briefing

Read `../../core/source-routing.md`, `../../core/safety-gates.md`, and
`../../core/output-schemas/daily_brief.md`.

Steps:
1. Define owner, scope, and date.
2. Rank customers by risk, urgency, stalled next action, and recent evidence.
3. Keep tracker updates lightweight: stage, risk, reminder, next action.
4. Produce a write plan before any tracker update.
